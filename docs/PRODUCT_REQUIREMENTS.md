# 随记账项目功能需求文档

> 当前功能事实与验收基线。后续需求变更先更新本文档，再进入开发。
> 战略背景和长期路线参见 `docs/product-strategy/prd-suishouji-2026-06-05.md`；如果战略 PRD 的进度状态与本文冲突，以本文和当前代码为准。

## 1. 产品概述

CashFlow（曾用名：随记账）是一款本地优先的 Android 智能记账 App，面向希望低成本管理个人消费的用户。

核心目标：

- 自动捕捉微信、支付宝支付信息。
- 支持相册导入支付截图并 OCR 识别。
- 所有自动来源先生成草稿，用户确认后入账。
- 支持手动快速记账。
- 用首页、账单、图表、预算帮助用户理解消费结构。
- 所有数据默认只保存在本机。

## 2. 目标设备与技术栈

- 目标设备：华为 Mate 70 Pro / HarmonyOS 4.x APK 使用场景。
- 开发语言：Kotlin。
- UI：Jetpack Compose + Material 3。
- 数据库：Room。
- 设置：DataStore。
- 异步：Kotlin Coroutines + Flow。
- 后台任务：WorkManager。
- OCR：Huawei ML Kit OCR。

## 3. 一级功能结构

1. 首页
2. 账单
3. 图表
4. 我的
5. 快速记账底部弹窗
6. 草稿编辑/确认弹窗

当前实现收敛为“4 个底部一级页面 + 顶部入口 + 二级功能页 + 弹窗”：

- 一级页面：首页、账单、图表、我的。
- 顶部入口：首页右上角搜索和 AI 对话，首页预算进度卡下方预算管理，首页待确认草稿下方 OCR 识别预览。
- 二级功能页：搜索筛选、AI 对话助手、预算管理、草稿审核、草稿详情、截图识别预览、数据同步、个人信息、设置、帮助与反馈、关于 CashFlow。
- 账户管理和分类管理不再占用当前可见主路径，后续如需维护，优先放回更轻量的二级页或独立管理入口。

## 5. 已实现功能

### 5.1 手动记账

用户点击底部「记账」或 FAB，打开底部弹窗。

已支持：

- 输入金额。
- 金额框支持 `+ - * /` 简单表达式实时计算，便于多笔小额消费一次合计。
- 选择支出/收入。
- 选择分类。
- 支出「其他」和收入「其他收入」在快速记账分类区作为「自定义」入口展示，点击后可新增当前收支类型的自定义分类并用于本次记账。
- 选择账户。
- 输入备注或商户。
- 保存后直接进入已确认流水。

验收标准：

- 金额为空或不合法时不保存。
- 金额表达式非法、计算结果小于等于 0 或除零时不保存。
- 金额以元输入，以分存储。
- 保存后首页和账单页数据刷新。

### 5.2 支付通知识别

通过 `PaymentNotificationListenerService` 监听支付通知。

已支持：

- 解析通知标题和内容。
- 提取金额、商户、交易类型、来源 App。
- 按包名推断账户。
- 生成待确认草稿。
- 通过指纹去重。
- 设置页提供“通知监听权限”入口，直接跳转系统通知监听设置页。
- 设置页「通知识别」开关在开启时会检查系统通知监听授权；未授权时引导到系统通知监听页，已授权时请求系统重新绑定监听服务。

规则：

- 只处理微信 `com.tencent.mm` 与支付宝 `com.eg.android.AlipayGphone` 通知。
- 非白名单 App 即使通知文本包含“支付/消费/收款/付款”等关键词，也不得生成草稿。
- 微信/支付宝普通聊天、群消息、订阅消息等没有有效金额与支付/收款语义的通知不得生成草稿，也不得生成 0 元草稿。
- 同一白名单 App 的通知处理需有轻量频率限制，异常高频通知不得持续刷入草稿。
- 微信/Tencent/WeChat -> 微信账户。
- Alipay/支付宝 -> 支付宝账户。
- 无法识别 -> 现金账户。
- 自动分类当前作用于所有自动来源的分类补全：通知草稿、OCR 草稿、AI 草稿在未明确识别分类时，都可使用商户/关键词推荐分类；关闭自动分类后仍保留草稿，但不额外补全分类。

验收标准：

- 自动识别结果不得直接确认为正式流水。
- 通知识别必须同时满足白名单包名、有效金额大于 0、支付/收款语义命中，才能生成草稿。
- 用户必须可以编辑、确认、忽略、删除。

### 5.3 相册导入支付截图

用户点击首页「待确认草稿」下方的「OCR 识别预览」入口，再选择支付截图。

已支持：

- 通过系统图片选择器选择图片。
- 将相册图片复制到 App 内部存储，避免 URI 权限失效。
- 插入 OCR_PENDING 草稿。
- WorkManager 调用 OCR。
- OCR 成功后回填金额、商户、分类、账户、原文。
- OCR 失败或 OCR 返回空文本时保留失败草稿，状态标记为 `OCR_FAILED`，不静默丢弃，也不长期停留在 `OCR_PENDING`。
- OCR 失败和空文本都视为本次识别结束，不自动进入 WorkManager 重试链路；后续重识别由用户手动触发。
- 用户可在编辑弹窗查看 OCR 原文。
- 识别预览页展示 OCR 草稿队列统计，包括待识别、已完成和失败数量，并在每条草稿上展示来源、置信度、指纹、图片 URI 和原文长度。
- OCR 原文诊断能力仍保留为辅助调试路径，但不占用首页入口。

验收标准：

- 导入后必须至少出现草稿。
- OCR 空结果也要可见，并展示为失败草稿。
- 同一相册图重复导入不能被 URI 去重直接吞掉。
- 粘贴 OCR 原文进行诊断时，不写入正式流水，也不绕过草稿确认机制。

### 5.4 截图监听

已具备服务入口：

- `ScreenshotMonitorService`
- 设置页「截图监听」开关。

目标行为：

- 监听截图目录新图片。
- 新截图自动生成 OCR 草稿。
- 结果进入待确认队列。
- 为兼容华为 Mate 70 Pro / HarmonyOS 真机启动限制，App 启动时不自动拉起截图监听前台服务；用户需在设置页手动开启，启动失败时开关应自动回退。
- 用户在设置页开启截图监听前，App 应按需请求读取图片与通知权限；用户拒绝授权时不开启服务，并保持开关关闭。
- 截图监听服务内部必须捕获前台服务启动、媒体库查询、内容观察者回调中的系统异常；权限缺失或 HarmonyOS 限制触发异常时，应记录日志、关闭设置开关并停止服务，不得导致 App 闪退。
- 服务启动或被系统回收后再次启动时，应补扫最近几秒新增截图，降低漏单概率。
- 当系统仅回调媒体总表 URI、空 URI 或非单条图片 URI 时，仍应从近期媒体记录中识别截图并入草稿。
- 媒体库近期补扫应避免依赖 ROM 不稳定的 SQL `LIMIT` 拼接，优先使用系统查询参数，降低真机启动后无结果或服务异常退出的概率。
- 截图判断需覆盖常见英文 `Screenshot`、`Screenshots`、`screen_shot`、中文“截图/截屏”等文件名或路径。
- 同一张截图在短时间内被多次媒体回调时，不得重复生成草稿，也不得把已完成 OCR 的草稿重置回待识别状态。
- 设置页截图监听开关开启时，先按需请求读取图片与通知权限；用户拒绝授权时保持开关关闭，并提示需要允许读取图片/通知权限后才能开启。
- 设置页同时提供“应用设置”入口，便于用户手动检查通知、存储等系统权限。

注意：

- Android/HarmonyOS 权限差异较大，后续必须在真机验证。

### 5.5 草稿队列

草稿来源：

- 支付通知。
- 相册 OCR。
- 截图 OCR。

草稿操作：

- 编辑。
- 确认。
- 忽略。
- 删除。
- OCR 草稿可重识别。

验收标准：

- CONFIRMED 才进入正式统计。
- IGNORED 不进入统计。
- DRAFT 必须在首页或账单页可见。

### 5.6 首页概览

已支持：

- 本月支出。
- 收入。
- 结余。
- 月预算进度。
- 预算剩余。
- 本月 Top 分类。
- 首页年月胶囊支持滚动选择年份和月份，概览、预算进度和 Top 3 分类跟随所选月份刷新。
- 首页 Top 3 支出分类卡片可点击查看所选月份该分类已确认支出明细、合计金额和笔数。
- 近期账单。
- 待确认草稿入口。

验收标准：

- 本月支出只统计当前月份 CONFIRMED 支出。
- 月预算为 0 时不显示进度警告。
- 月预算超过 80% 预警，超过 100% 超支。

- 账单页顶部年月胶囊支持滚动选择年份和月份。
- 月份左右切换。
- 每日支出格子。

- 引线标签必须显示分类名称和占比，且不能遮挡图表主体。
- 月度趋势曲线必须基于已确认流水的月度统计，正增长绿色、负增长红色；支出月度曲线基于已确认支出统计，支出增加红色提示、支出下降绿色提示。

### 5.9 预算体系

已支持：

- 设置本月总预算。
- 设置分类月预算。
- 首页显示总预算使用情况。
- 图表页显示分类预算使用情况。
- 预算预警开关。
- 本月总预算达到 80% 与 100% 时发送本地系统提醒。

规则：

- 预算以元输入，以分存储。
- 0 或空表示未设置。
- 分类预算只对支出分类生效。
- 设置页默认以金额行展示本月总预算，点击预算行后展开输入与保存操作。
- 预算提醒默认开启，仅在已授权通知权限时生效。
- 预算提醒标题和正文已统一为中文文案，与 App 其他通知提示保持一致。
- 同一个自然月、同一预算金额下，80% 预警和 100% 超支提醒各最多触发一次。

后续待做：

- 超预算高亮和提醒记录。
- 预算按月份保存历史。

已支持：

- 默认账户。
- 自定义账户新增。
- 自定义账户改名。
- 自定义账户颜色选择，使用预设色板写入 `colorHex`。
- 自定义账户图标选择，使用预设图标写入 `iconKey`。

- 系统账户包括微信、支付宝、银行卡、现金。

## 6. 自动识别与去重规则

### 6.1 通知去重

使用来源包名、时间桶、金额、商户、原始文本摘要构建指纹。

目标：

- 避免同一支付通知被重复解析为多条草稿。

### 6.2 OCR 去重

优先级：

1. 如果 OCR 文本中存在订单号/交易单号，使用交易号构建指纹。
2. 如果存在交易时间，使用金额 + 商户 + 时间构建指纹。
3. 兜底使用金额 + 商户 + 文本 hash。

相册导入特殊规则：

- 相册导入允许同一图片再次生成草稿，避免用户重试时被吞。
- 如果和已存在草稿重复，可复用或更新草稿。
- 如果和已确认流水重复，应保留草稿给用户判断。

## 7. 分类规则

内置分类当前包括：

- 餐饮
- 咖啡茶饮
- 外卖
- 交通
- 打车
- 购物
- 日用
- 娱乐
- 居住
- 医疗
- 教育
- 通讯
- 旅行
- 服饰
- 数码
- 生活缴费
- 转账
- 其他

内置收入分类当前包括：

- 工资
- 炒股收入
- 奖金
- 副业收入
- 退款返现
- 其他收入

自动分类目标：

- 基于商户和关键词推荐分类。
- 用户确认后未来可沉淀为自定义规则。
- 当前已实现的自动分类用于通知草稿、OCR 草稿和 AI 草稿的分类补全，不会把已识别出的分类强行改写，也不会自动将草稿直接确认为正式流水。
- 手动记账页面也会基于备注/商户给出分类建议，用户可一键采纳推荐分类。

分类管理已支持：

- 支出分类和收入分类分别新增自定义分类。
- 工资为系统固定收入分类，不可编辑删除。
- 炒股收入、奖金、副业收入、退款返现、其他收入为系统收入分类。
- 自定义分类改名。
- 自定义分类颜色选择，使用预设色板写入 `colorHex`。
- 自定义分类图标选择，使用预设图标写入 `iconKey`。
- 自定义分类删除。
- 系统分类不可编辑删除。
- 支出分类和收入分类各自支持排序模式，可上移、下移、置顶、置底；排序结果写入 `sortOrder` 并立即影响设置页、快速记账、编辑流水和图表展示顺序。
- 删除分类不删除历史流水，历史流水继续显示 `categoryNameSnapshot`。
- 手动记账和编辑流水的分类选择必须与分类管理同步。
- 支出流水只显示支出分类，收入流水只显示收入分类。

## 9. 隐私与数据安全

当前原则：

- 数据本地存储。
- 不上传服务器。
- OCR 在本机侧调用 Huawei ML Kit。
- 原始 OCR 文本保留在本机，用于用户复核。
- App 关闭系统自动备份，并在备份/设备迁移规则中排除数据库、DataStore、认证文件等敏感本地数据。
- 登录 Token、userId、username、email 存入 Android Keystore 保护的加密本地存储；旧版 DataStore 明文 Token 首次读取时迁移到加密存储并清除旧认证字段。
- Debug 构建保留局域网 HTTP 后端地址用于真机体验；Release 构建的 API baseUrl 必须为 HTTPS，否则构建失败。
- 网络请求日志仅 Debug 使用 BODY 级别；Release 关闭 HTTP 日志输出。
- Release 构建启用 R8 混淆，并保守保留 Retrofit、Room、Gson、WorkManager Worker 与 Huawei ML Kit 反射所需类型，避免混淆破坏真机功能。
- 本地体验 APK 分支启用 `BuildConfig.LOCAL_PREVIEW`：保留登录/注册、AI 对话、数据同步和个人信息入口，但点击后只展示“本地体验版暂不开放登录、AI 和云同步；当前可使用本地记账、账单、图表和识别草稿。”，不得发起真实网络请求。
- 本地体验 APK 的数据仅保存在本机，适合通过私有 GitHub Release 附件供受邀协作者侧载体验，不作为正式生产发布包。
- 分支分发时应固定生成两份稳定 APK 产物：`app-debug-main.apk` 与 `app-debug-local-preview.apk`，不要长期依赖会被覆盖的 `app-debug.apk` 作为交付文件。

后续可选：

- 数据导出前身份验证。
- 本地数据库加密。
- JSON/CSV 备份与恢复。

### 5.11 远程后端与认证（Phase 1+2 新增）

后端为 Spring Boot 3.2 + MySQL 8.0 + Redis 7，提供 REST API 服务。

认证功能：
- 邮箱+密码注册。
- 邮箱+密码登录，返回 JWT Token。
- Token 存储于 Android Keystore 保护的加密本地存储；旧版 DataStore Token 读取时自动迁移并清除旧认证字段，保留上次登录邮箱用于回填。
- 未登录时允许继续使用本地记账、账单、图表、我的/设置等离线功能。
- 云同步与 AI 对话仅对已登录用户开放；未登录时引导用户进入登录/注册页。
- 本地体验 APK 中登录、注册、AI 对话、云同步、个人信息等联网入口全部降级为统一提示，不进入真实登录页或网络流程。
- 我的页和设置页都提供登录入口和退出登录；退出后必须清除本地认证并打开登录/注册页，但可保留上次登录邮箱用于回填，用户仍可选择继续使用本地版。
- App 主界面订阅本地 Token 状态恢复登录态；云同步或 AI 对话接口返回 401/403 时，自动清除本地 Token 并回到未登录状态，提示用户重新登录。

远程 API 同步：
- 本地优先策略，写入 Room 后立即返回。
- Retrofit + OkHttp 访问后端 API。
- SyncManager 提供 upload 和 download 方法用于增量同步。
- 同步请求携带 `X-CashFlow-Sync-Version` 协议版本头，当前版本为 `2`。
- 同步上传只包含 `CONFIRMED` 流水，并排除 `TRANSFER`；DRAFT/IGNORED 草稿不会上传到云端。
- 后台同步必须上传成功后才执行下载；上传失败时直接停止并返回失败状态，不误继续合并远端数据。
- 同步页默认的“立即同步”按上传成功后再下载的顺序执行，并持久化最近同步时间、同步状态和失败原因。
- 同步下载写入时先按 `id` 查找，再按 `sourceFingerprint` 查找；已存在记录执行合并更新，避免同一流水重复插入。
- 下载合并以 `updatedAt` 较新者为准，本地较新的记录不得被远端旧数据静默覆盖。
- SyncManager 遇到 Token 过期时不得继续重试失效 Token，应返回“登录已过期，请重新登录”并触发登录态回退。
- 后端同步接口使用 Android 账单 JSON 形态，显式映射 `LedgerEntryEntity.id` 与服务端 `client_id`，并保留分类/账户名称快照与 OCR 相关字段，便于移动端直接解析。
- 后台同步通过 WorkManager 以唯一任务执行，避免独立协程脱离系统后台调度边界；后续可在此基础上扩展定时同步。

### 5.12 AI 对话（Phase 3 已实现 - DeepSeek API）

- AI 对话功能已实现；当前入口放在首页右上角，「我的」页保留二级入口，不进入底部导航。
- ChatScreen 当前收敛为单会话聊天区、刷新按钮、输入框和结构化草稿卡片，不再展示多会话 chip 和冗长说明。
- 后端 AiChatService 接入 DeepSeek API（deepseek-chat 模型），对话上下文存储在 Redis。
- Function Calling 已实现 4 个工具：
  - add_transaction：从自然语言识别并记录账目草稿
  - query_transactions：查询历史流水
  - get_spending_summary：消费统计汇总
  - get_category_by_name：模糊匹配分类名
- AI 识别到自然语言记账意图后，后端优先用确定性解析器返回 `PENDING_CONFIRMATION` 结构化草稿；未命中时再走 DeepSeek Function Calling，不直接写入服务端正式流水。
- Android 端对“帮我记一笔/补一笔/花了/买了/支付/付款/消费”等明确记账输入应先进行本地确定性解析；能识别金额时立即在当前聊天中生成待确认草稿卡片，不等待 LLM 自由回复。金额应兼容“650元”“650块”“¥650”以及“机票650 微信”这类无单位但语义明确的记账表达，并避免把日期数字误判为金额。
- Android 端在 AI 消息下展示极简“待确认草稿”卡片，只显示消费分类、金额、时间，以及“取消”“确认入账”两个按钮。
- Android 端需要兼容后端返回的 `tool_calls` 数组、`PENDING_CONFIRMATION` 草稿对象、工具调用 `content` 参数和普通自然语言金额文本，并稳定渲染为同一张确认卡片。
- Android 端从普通 AI 文本渲染草稿卡片时，应优先读取结构化字段和 Markdown 表格中的“分类/日期/账户”等值，再做关键词推断；话费、手机费、流量费、宽带等通讯类表达不得误归为咖啡茶饮；未识别分类时应落到“其他/其他收入”，不得默认使用第一个分类造成误判。
- Android 端必须保留 AI 草稿中的交易日期/时间字段，兼容 `occurredAt`、`occurredAtMillis`、`transactionTime`、`dateTime`、`date` 等结构化字段以及 `2026-06-08`、`2026-06-08 21:30`、ISO 时间、5号、6月5号、昨天/前天、上周五、晚上等相对或口语日期；确认入账后不得把历史账单默认落到今天。
- Android 端发送 AI 对话消息时必须附带本地 Room 中已确认流水上下文；AI 查询账单、消费汇总、收入汇总时应优先使用这份本地已确认账本，而不是仅查询远程数据库，避免未同步数据回答为 0。对“本月消费多少”“今天支出多少”等统计类问题，Android 端可直接基于本地 Room 生成回答，保证与首页、账单、图表同源。
- AI 草稿卡片确认时必须先在本地 Room 写入 `reviewState = DRAFT` 的草稿，再立即确认入账，确保首页、账单和图表通过 Flow 自动刷新。
- AI 草稿确认需具备幂等性：同一 `clientId` 或 `sourceFingerprint` 重复确认时不得重复生成流水，已存在草稿应确认，已确认流水不应被重复覆盖。
- AI 草稿重复确认时，若本地已存在 `CONFIRMED` 记录，应直接返回数据库中的确认结果，不得用外部传入的草稿字段覆盖已确认记录。
- AI 草稿卡片重新进入对话页时必须读取本地 Room 已确认流水状态；已入账的卡片展示“已入账”并禁用重复确认。
- 后端确定性解析器应优先识别带金额单位或支付/收入动词附近的金额，避免把日期中的年份、月份、时间误判为金额，并尽量从自然语言中推断账户。
- 用户点击“取消”后只关闭当前草稿卡，不写入本地流水。
- 草稿卡片不再暴露编辑态细项，先按极简确认路径交付，后续再视真机反馈补充编辑入口。
- AI 对话请求遇到 Token 过期时，自动清空本地登录态和当前会话缓存，并提示重新登录后继续使用。
- AI 对话页右上角刷新按钮用于清空当前聊天框并开启新对话，后续消息进入新的会话上下文。
- API Key 通过 deepseek.api-key 配置（支持环境变量 DEEPSEEK_API_KEY）。
- 待优化：AI 草稿字段补全、多轮追问和更强的日期编辑体验。

## 13. 截图识别升级约定

- 截图 OCR 需要同时支持“单笔详情页”和“多笔列表页”两类结构。
- 单笔详情页优先提取页面中央金额、商户名、支付时间、交易单号。
- 单笔详情页中“商户简称/商户全称”等字段被 OCR 拆成多行时，应跳过“简称/全称”等标签碎片，继续向后提取真实商户名。
- 账单列表页需要能按条拆分成多个待确认草稿，不能把整页只识别成一笔。
- 账单列表页 OCR 若把商户/时间与右侧金额读成分离的两段文本，应按出现顺序将交易行与负金额配对，恢复多笔草稿。
- 微信/支付宝列表页中的“商户名 + 05-17 12:34”同一行结构应可直接识别，并向后匹配对应负金额。
- 微信/支付宝列表页中的“商户名 + 昨天 08:15 / 前天 21:03”同一行相对时间结构应可直接识别，并向后匹配对应负金额。
- 支付宝“搜索交易记录/订单列表”截图应支持一图多消费识别，兼容“昨天23:13”“05-1712:34”等无空格时间格式，并按列表顺序生成多条支出草稿。
- 详情页商户名允许包含数字、连字符和字母组合，例如“7-ELEVEN便利店”“MANNER COFFEE 021 店”，不能因为包含数字被误判为非商户。
- 支付宝列表中的“交易关闭”和正数金额行暂不导入为消费草稿，后续如支持退款/收入导入再单独扩展。
- 列表页中的月度汇总、红包抵扣、奖励、积分等辅助字段不能误当成交易金额。
- OCR 识别到多笔结果时，仍然必须全部先进入草稿队列，再由用户确认。
- 重复导入或重识别同一张列表截图时，如果主记录已存在，也应补齐尚未生成的后续草稿。
